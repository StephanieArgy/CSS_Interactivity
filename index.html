Enhancing Interactivity with CSS

If you want to add interactivity to your web pages without JavaScript or jQuery, you still can do a lot with CSS:

Pseudo-Classes

In your CSS, you can apply different styles to an HTML element, so that a user's interaction with that element will change its appearance. For example, think of hyperlinks that change style when you mouse over them, click on them, or have visited the link:

Demo Link

You create these different styles in CSS with "pseudo-classes," which are keywords used after a CSS selector to specify a particular state. These pseudo-classes start with a colon and follow right after the element name. Here are examples of four of the most common, shown here used for hyperlinks:

:link

/* This is for a link that hasn't been visited yet */

a:link {
Your styles go in here.
}

:visited

/* This is for a link that the user has visited */

a:visited {
Your styles go in here.
}

:hover

/* This is for a link that the user is hovering over with a mouse -- and it does NOT work on touch-screen devices like iPads and iPhones */

a:hover {
Your styles go in here.
}

:active

/* This is for an element being activated (for example, a link at the moment that it's clicked on) */

a:active {
Your styles go in here.
}

You can apply pseudo-classes to all sorts of elements -- for instance, if you wanted a paragraph to turn orange and be underlined when a user hovered over it, you could do this:

p:hover {
color: orange;
text-decoration: underline;
}

And then if someone were to put their cursor over your paragraph, it would behave like this:

You can also use pseudo-classes on a button. If you wanted the button to appear to pop up a little and cast a shadow, you'd first set the styles for the button in its normal state, then create a :hover pseudo-class to make the button change when someone mouses over it:

button {
background: #99CCFF;
border: 2px solid black;
font-size: 1.2rem;
padding: 5px;
}

button:hover {
background: #ADD6FF;
box-shadow: 5px 5px 5px #888888;
position: relative;
bottom: 2px;
right: 2px;
}

CSS Animation

CSS3 makes it possible to animate most HTML elements without the use of Flash, JavaScript of jQuery. You can read about the basics here:

  • W3 Schools: CSS3 Animations
  • CSS Animation for Beginners
  • CSS-Tricks: Animation
BUT...a cautionary note! Some browsers still require -webkit- prefixes for CSS3 animation, so be sure to go to caniuse.com or shouldiprefix.com to make sure you've added what you need to.

If you wanted to, you could change the color of a div:

div {
width: 100%;
height: 200px;
animation: pulse 5s infinite;
animation-direction: alternate-reverse;
}

@keyframes pulse {
0% {
background-color: #001F3F;
}
50% {
background-color: red;
}
100% {
background-color: #FF4136;
}
}

Or you could use this code...

h1 {
animation-duration: 3s;
animation-name: slidein;
}

@keyframes slidein {
from {
color: green;
margin-left: 100%;
width: 300%;
}

to {
color: red;
margin-left: 0%;
width: 100%;
}
}

...to make text slide across the screen and turn from green to red, like this:

Microinteractions

In thinking about what to do with all of these CSS capabilities, one more thing to consider is the concept of "microinteractions." What are you trying to help the user do? How can your CSS interactions and animations help make that easier and more sensible for him or her?
